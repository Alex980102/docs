Este módulo permite a los asesores cargar sus comprobantes fiscales directamente desde el dashboard para validar y gestionar el cobro de sus comisiones.  
Automatiza la revisión de archivos y aplica reglas fiscales para garantizar que los documentos estén correctos.

---

## Acceso al módulo

Para ingresar:

1. Dirígete a **Dashboard > Canales**.
2. Busca al asesor correspondiente.
3. Haz clic en los **tres puntos (···)** y selecciona **Ver recibos**.
4. En el listado, selecciona el mes correspondiente.
5. Da clic en **Acciones > Subir factura [RAZÓN SOCIAL]**.

## Carga de factura

Una vez dentro, verás precargada la información fiscal vinculada a la comisión del asesor:

- Razón social
- RFC
- Domicilio fiscal
- Uso de CFDI
- Concepto
- Unidad de medida
- Cantidad
- Monto a cobrar

Deberás subir:

<ul>
  <li>PDF de la factura</li>
  <li>Archivo XML</li>
  <li>Opinión de cumplimiento del SAT vigente</li>
</ul>

<Note>
  Toda la información está enlazada al periodo y monto autorizado por el área de
  comisiones.
</Note>

---

## Validaciones automáticas

Al presionar **Validar**, el sistema revisará automáticamente los siguientes puntos del XML y del archivo de cumplimiento:

**En XML:**

<ul>
  <li>RFC emisor y receptor</li>
  <li>Régimen fiscal</li>
  <li>Folio fiscal (UUID)</li>
  <li>Fecha y hora de emisión</li>
  <li>Uso del CFDI</li>
  <li>Moneda, método y forma de pago</li>
  <li>Claves SAT (producto/unidad)</li>
  <li>Subtotal, impuestos, total</li>
</ul>

**Tipo de comprobante:**

<ul>
  <li>PUE: debe pagarse en el mismo mes de emisión</li>
  <li>PPD: puede pagarse hasta 27 días después de su emisión</li>
</ul>

**Opinión de cumplimiento del SAT:**

<ul>
  <li>Nombre o razón social</li>
  <li>Estado debe incluir la palabra “POSITIVO”</li>
  <li>Debe tener fecha del mes actual</li>
</ul>

---

## Flujo del sistema

1. El asesor entra al módulo y selecciona un periodo disponible.
2. Se muestra el monto autorizado para facturar.
3. Sube el PDF, XML y opinión de cumplimiento.
4. El sistema valida automáticamente los archivos.
5. Si hay errores, se indica el motivo exacto.
6. Si es correcto, queda listo para pago.

---

## Gestión de ajustes

Los ajustes se gestionan por el área de comisiones y se cargan mediante CSV.  
Estos pueden sumar o restar al balance del asesor.

| Campo               | Descripción                         |
| ------------------- | ----------------------------------- |
| Fecha de aplicación | Fecha en que se reflejará el ajuste |
| RFC                 | Asociado al asesor                  |
| Monto               | Positivo o negativo                 |
| Motivo              | Justificación breve                 |

---

## Reglas generales

- Solo se permite **una factura por RFC y por corte**.
- El sistema validará contra el monto autorizado por comisiones.
- Si el comprobante es inválido, el asesor verá un mensaje y podrá corregirlo.
- Las facturas se evalúan en tiempo real, sin intervención humana.
- Los errores se deben resolver antes del pago; si persisten, se puede abrir un ticket.
